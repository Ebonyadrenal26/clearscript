# Launch posts

Drafts for the public-launch wave once the user has run clearscript on a real transcript and feels confident about quality. Pre-tested copy so you're not writing from scratch on launch day.

| Post | Audience | Tone |
|---|---|---|
| [`x-thread.md`](./x-thread.md) | Twitter / X | 8 tweets, story-led, ends with the link |
| [`hn-post.md`](./hn-post.md) | Hacker News | "Show HN", architecture-curious tone, with FAQ replies |
| [`zhihu-post.md`](./zhihu-post.md) | 知乎 / 微信公众号 / V2EX | 中文长文，VC + 工程师都能看 |

## Recommended order

1. **HN first** (Tuesday or Wednesday morning Pacific). The most technical audience — early issues / PRs / star spike.
2. **X thread same day**, in the evening (Pacific) so it cross-pollinates with the HN traffic. Pin the thread.
3. **知乎/微信 the next day** — the Chinese tech / VC community has the highest concentration of users with this exact pain (ASR cleanup of 飞书妙记 output, etc.). Don't bury this — for clearscript it might be the highest-converting channel.

## Don't ship until

- [ ] You've run clearscript on at least 3 of your own real transcripts and trust the output
- [ ] A demo GIF (~5 seconds: paste → Run → diff highlights) lives at `docs/assets/demo.gif`
- [ ] Issue templates work (try filing one yourself)
- [ ] The first comment on each platform has been pre-thought (HN: respond to the most likely "but why not X?" within an hour or it dies)
- [ ] You're ready to absorb 30+ issues in the first week

## After launch

The library is the moat. So: ship one user-facing improvement per week for the first month. Stagnant repos die. The visible commit cadence is itself marketing.
